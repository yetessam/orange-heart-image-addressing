Findability Protocols for Web Content Addressing

Humans use an infinite number of different modes of addressing from sentimental to scientific to help us locate things.

Findability Protocols

What is an Findability Protocol? It is a system that's generally shared by one or more actors to make things findable. Let's group together all of these diverse systems together as "findability protocols." Any of the following could be considered examples of findability protocols:
  • Street addresses
  • Physical landmarks
  • Dewey decimal system
  • Geographic systems such as longitude and latitude
  • Sonar

Web Content Addressing Methods

The web uses a combination of findability protocols to address and retrieve content. The key addressing protocols include:
  • Content Addressable Identifiers (CIDs)
  • Uniform Resource Locators (HTTP URLs)
  • Domain Names
  • Metadata for Accessbility and Search Engine Indexing

The current website uses content-based and location-based addressing, along with accessiblity metadata.

Content Addressing

When you take any digital asset, say an image, and process it with a cryptographic hash, or function, the result is a unique series of numbers and letters for this exact image.

This result is a content address ID (CID). These CIDs that result from cryptographic hash are a form of the image's address. as a location address. If I share the following link, it lives in multiple places on the web but all of the following links are guaranteed to be the exact same, unaltered image.

As this is a globally unique ID, if I request this image, can be retrieved from any number of servers. Its authenticity can be easily verified by running the image through a hash and verifying that it is indeed the same image.

Another way to look at CIDs is to consider them as a text-based and unique property of the content object.

If you've published a book and have a different version of the book, its CID will change. In that case, you can create specific links to the original version of the book as well as version two.
