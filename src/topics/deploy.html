Deployment

Deploying to the dweb for the enterprise requires content to be published, pinned, and hosted on decentralized storage.

Most enterprise content publishing processes remain the same when moving to the dweb.

The main charactertisic is the use of decentralized storage using IFPS and ensuring content is pinned to ensure its availability.

Fleek provides a unique IPFS gateway which allows users to access the content through URLs rather than content identifiers. As a result, user access the site through https://delicate-glitter-6121.on-fleek.app/ . At this point, you can apply your domain name.

When changes were merged into my Github prod branch, Fleek deployment started.

Fleek deployed the content to IPFS, generateed a new IPFS hash (CID), and pinned it.

Multiple platforms and services can publish, pin, and host the content to the decentralized IPFS storage. IPFS has its own solutions including a CLI that can be used to deploy to dweb. Generative AI, Open AI in particular, is adept at identifying emergent dweb deployment options. One final thing to note is that we could be publishing dapps (decentralized web apps), regular apps or other digital assets using the same stack.
