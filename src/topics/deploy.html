Deployment

Multiple platforms and services can publish, pin, and host the content to the decentralized IPFS storage.

This site's content publishing is automated by connected Fleek to a Github CI/CD. Whenever changes are pushed to the prod branch, Fleek runs a deployment.

The general process is that Fleek deploys the content to IPFS, generates a new IPFS hash (CID), and pins the content to ensure that it remains accessible.

Fleek provides a unique IPFS gateway URL (e.g., https://https://delicate-glitter-6121.on-fleek.app/). It supports providing a custom domain or a static IPNS namespace identifiers. Fleek does support building on Fleek, but I'm building on Git and only deploying through Fleek.
